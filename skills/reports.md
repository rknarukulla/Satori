# Skill: HTML Report Generation

Loaded when the user asks for a dashboard, report, or shareable output.

---

## When to Generate a Report

- User explicitly asks: "give me a dashboard", "create a report", "make something I can share"
- After a weekly summary, offer: "Want me to turn this into a visual report?"
- After a content calendar, offer: "Want me to save this as an HTML file?"
- Never generate a report unsolicited — always confirm first

---

## Report Types

### 1. Weekly Performance Dashboard
Filename: `reports/weekly-{YYYY-MM-DD}.html`

Sections:
- Header: account name, week range, "Generated by Satori"
- Summary row: reach, impressions, new followers, profile visits (4 stat cards)
- Reach trend: line chart (daily reach over 7 days) using Chart.js
- Post performance: bar chart (reach per post, colored by media type)
- Top 5 posts table: thumbnail placeholder, caption excerpt, reach, saves, engagement rate
- One insight callout box: the most important finding in plain English

### 2. Content Calendar
Filename: `reports/calendar-{YYYY-MM}.html`

Sections:
- Header: account name, month, account type badge (Creator / Business)
- 4-week table: color-coded by format (Reel = blue, Carousel = green, Image = orange)
- Strategy notes block: 3 bullet points on approach for the month
- Hashtag clusters: listed by week

### 3. Post Analysis Report
Filename: `reports/post-{media_id}-{YYYY-MM-DD}.html`

Sections:
- Post metadata: type, date, caption
- Metric comparison table: this post vs account average
- Percentile badges: "Top 10% for saves", etc.
- Visual analysis block (if Llama Vision ran)
- Comment sentiment summary (if available)
- Recommendation: replicate or avoid, and why

## HTML Template Rules

**Always self-contained:**
- No external CSS files
- Chart.js loaded from CDN: `https://cdn.jsdelivr.net/npm/chart.js`
- All styles inline in `<style>` tag
- No server, no backend — opens directly in any browser

**Design system:**
```css
--bg: #fafaf8;
--ink: #1a1a1a;
--muted: #6b6b6b;
--line: #e0e0dc;
--accent: #2d5be3;       /* Creator: blue */
--green: #1a7f5a;        /* Business: green */
--orange: #c75000;       /* Warning / Reel */
font-family: 'Georgia', serif for body; 'Courier New', monospace for labels/data
```

Use `--accent` (blue) for creator accounts, `--green` for business accounts as the primary color throughout.

**Chart.js patterns:**

Line chart (reach trend):
```javascript
new Chart(ctx, {
  type: 'line',
  data: {
    labels: {date_labels_array},
    datasets: [{
      label: 'Daily Reach',
      data: {reach_values_array},
      borderColor: '#2d5be3',
      backgroundColor: 'rgba(45,91,227,0.08)',
      tension: 0.3,
      fill: true
    }]
  },
  options: { plugins: { legend: { display: false } }, scales: { y: { beginAtZero: true } } }
});
```

Bar chart (post performance):
```javascript
new Chart(ctx, {
  type: 'bar',
  data: {
    labels: {post_labels},
    datasets: [{
      label: 'Reach',
      data: {reach_values},
      backgroundColor: {color_per_media_type}
    }]
  }
});
```

---

## Writing the File

After generating the HTML:

1. Write to the appropriate `reports/` path using the Write tool.
2. Tell the user: "Your report is ready — open `reports/{filename}` in any browser. You can share the file directly."
3. Offer to generate a different report type if relevant.

---

## Report Data Sourcing

Always use already-fetched data from the current session if available — don't re-fetch just to generate a report. If data isn't in the session yet, fetch it first, then generate.

For weekly reports: use the same data pulled during the weekly summary conversation turn. The report is a visual rendering of that data, not a new data fetch.
